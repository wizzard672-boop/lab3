# Отчёт по решению задач варианта
### Задача 1
Игорь составляет таблицу кодовых слов для передачи сообщений, каждому сообщению соответствует своё кодовое слово. В качестве кодовых слов Игорь использует трёхбуквенные слова, в которых могут быть только буквы Ш, К, О, Л, А, причём буква К появляется ровно 1 раз. Каждая из других допустимых букв может встречаться в кодовом слове любое количество раз или не встречаться совсем. Сколько различных кодовых слов может использовать Игорь?
## Описание проделанной работы

### Решение задачи 1
**Анализ условия:**
- Длина слова: 3 буквы
- Алфавит: Ш, К, О, Л, А (5 букв)
- Ограничение: буква К встречается ровно 1 раз

**Ход решения:**
1. Буква К может занимать одну из трёх позиций в слове — 3 способа
2. На каждой из оставшихся двух позиций может быть любая из 4 букв (Ш, О, Л, А)
3. По правилу произведения: 3 × 4 × 4 = 48

**Ответ:** 48 различных кодовых слов

### Задача 2
Значение арифметического выражения 3·4³⁸ + 2·4²³ + 4²⁰ + 3·4⁵ + 2·4⁴ + 1 записали в системе счисления с основанием 16. Сколько значащих нулей содержится в этой записи?

### Решение задачи 2
**Ход решения:**
1. Вычисляем значение арифметического выражения с помощью программы
2. Переводим полученное число в шестнадцатеричную систему счисления
3. Подсчитываем количество символов '0' в шестнадцатеричной записи

**Анализ выражения:**
- Выражение содержит степени числа 4
- В шестнадцатеричной системе 16 = 4², поэтому каждая шестнадцатеричная цифра кодирует 2 разряда в четверичной системе
- Программа вычислила точное значение и выполнила перевод

### Задача 3
Напишите программу, которая перебирает целые числа, большие 600 000, в порядке возрастания и ищет среди них такие, среди делителей которых есть хотя бы одно число, оканчивающееся на 7, но не равное 7 и самому числу. Необходимо вывести первые 5 таких чисел, и наименьший делитель, оканчивающийся на 7, не равный 7 и самому числу.

### Решение задачи 3
**Алгоритм:**
1. Начинаем перебор с числа 600 001
2. Для каждого числа ищем все делители до корня из числа
3. Проверяем каждый делитель и парный ему делитель на соответствие условиям:
   - Оканчивается на 7 (последняя цифра 7)
   - Не равен 7
   - Не равен самому числу
4. Среди подходящих делителей выбираем наименьший
5. Продолжаем перебор, пока не найдём 5 чисел

**Оптимизация:**
- Перебор делителей до √n сокращает время выполнения
- Используется эффективная проверка последней цифры через остаток от деления на 10
